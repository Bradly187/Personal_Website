Identity Verification Pipeline

// Architecture
S3 Document Bucket
Encrypted, private bucket storing patient/therapist images. EventBridge triggers workflow on selfie upload.
EventBridge Rules
Decoupled event routing — detects _selfie.png or _therapist_selfie.png uploads and starts the correct state machine.
Step Functions (Patient)
Orchestrates 3 parallel branches: face comparison, license OCR, and insurance extraction. Merges results and stores verification.
Step Functions (Therapist)
Parallel branches: face comparison, medical license verification, and medical license OCR. Same merge-and-store pattern.
CompareFaces Lambda
Rekognition face match between selfie and driver's license photo. Returns similarity score and match boolean.
ExtractLicense Lambda
Textract analyze_id to OCR driver's license — extracts name, DOB, address as structured data.
ExtractInsurance Lambda
Textract analyze_document with Queries to read insurance card — carrier, policy number, group ID.
VerifyMedicalLicense Lambda
Calls an external medical board registry to validate therapist license number and status.
StoreVerification Lambda
Aggregates parallel results, writes to DynamoDB, and POSTs verification to the ClinicScheduler API.
DynamoDB + SNS
Patient/Therapist verification tables with auto-scaling. SNS alerts staff on failures or manual review needed.
// Workflow

Upload Images → S3 EventBridge Trigger → Step Functions → Parallel (Face Match + License OCR + Insurance OCR) → Store Verification → DynamoDB + Clinic API
on failure → SNS Alert to Staff

// Security
Encrypted Storage
S3 bucket uses AES-256 server-side encryption for all patient images.
HTTPS-Only Policy
Bucket policy denies any request not using secure transport (aws:SecureTransport).
90-Day Lifecycle
Raw images auto-expire after 90 days — processed data lives in DynamoDB.
Auto-Scaling DynamoDB
Read/write capacity scales 2–20 units based on 70% utilization target.
X-Ray Tracing
Active tracing on all Lambda functions and state machines for observability.
Point-in-Time Recovery
DynamoDB tables have PITR enabled for data protection and recovery.
